## Coding Standards

*   **Naming Conventions:**
    *   Variables, Functions, Methods, Parameters: `camelCase` (e.g., `userId`, `getUserData`)
    *   Classes, Enums, Typedefs, Extensions: `PascalCase` (e.g., `UserService`, `UserRole`)
    *   Libraries, Packages, Directories, Source Files: `snake_case` (e.g., `user_service.dart`, `lib/models/`)
    *   Constants: `camelCase` (often prefixed with `k`, e.g., `kMaxUsers`)
*   **Formatting:** Code formatting is enforced by `dart format`. Run `dart format .` or configure your IDE to format on save.
*   **Linting:** Code quality, style, and potential errors are checked by the Dart analyzer using rules defined in `analysis_options.yaml`. Adhere to the default recommended Flutter rules.
*   **Comments:** Use `///` for documentation comments on public APIs (classes, functions, etc.). Use `//` for implementation comments, explaining complex logic or reasoning.
